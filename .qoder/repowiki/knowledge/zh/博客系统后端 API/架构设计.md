- **分层架构**：采用 `api/v1` (Handler) -> `model` (Service/DAO) -> `utils` (Infrastructure) 的分层设计。路由在 `routers/routers.go` 统一注册，通过 `middlewares` 实现 JWT 鉴权与速率限制。
- **数据持久化**：使用 GORM 作为 ORM，支持 MySQL 和 SQLite 自动切换（见 `model/DB.go`）。模型层封装了业务逻辑（如密码加密、标签解析），Handler 层仅负责参数绑定与响应组装。
- **配置管理**：通过 `utils/setting.go` 加载 YAML 配置，支持环境变量注入与运行时重载。
- **静态资源**：路由层直接映射 `uploads` 及前端静态目录，实现文件直连访问。
